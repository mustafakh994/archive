# شرح مشكلة مرفقات الموبايل (404) والحل

## ما المشكلة؟

عند رفع صورة أو ملف من **تطبيق الموبايل** كان يُحفَظ في البيانات رابط يفتح على نفس موقع الواجهة، مثلاً:

- `https://archive.hamaprov.net/اسم-ملف.jpg`
- أو `https://archive.hamaprov.net/api/files/download/معرّف-الملف`

موقع الواجهة (Next.js) **لا يخزّن** ملفات الـ API بهذه المسارات، ولم يكن لديه مسار جاهز يمرّر الطلب إلى الخادم الخلفي (ASP.NET). النتيجة: المتصفح يعرض **404 | This page could not be found**.

من **الداشبورد** غالباً يُستخدم رابط صحيح (مثل مسار `/api/upload` أو رابط يشير مباشرة إلى الـ API)، لذلك لم تظهر نفس المشكلة.

---

## كيف تم الحل؟ (باختصار)

1. **في الواجهة (Next.js)**  
   إضافة مسارات (Proxy) تحت `/api/files/download/...` تقوم بإعادة توجيه الطلب تلقائياً إلى عنوان الـ API الحقيقي (`NEXT_PUBLIC_API_URL`) وتعيد الملف للمستخدم.  
   بهذا يعمل الرابط حتى لو كان مكتوباً على دومين `archive.hamaprov.net`.

2. **في الـ API (ASP.NET)**  
   إضافة مسار تنزيل إضافي بالاسم المخزَّن على القرص (`download/by-name/{fileName}`) للتوافق مع الروابط القديمة التي تحتوي على اسم الملف فقط (مثل `guid.jpg`) بدل المعرّف فقط.

3. **في صفحات الداشبورد**  
   دالة تطبيع للروابط (`normalizeAttachmentUrl`) تحوّل بعض أشكال الروابط القديمة إلى عنوان تنزيل صحيح على الـ API عند العرض أو التصدير.

---

## الملفات التي تم التعديل عليها أو إضافتها

### مشروع `aspforms` (ASP.NET API)

| الملف | ماذا تغيّر؟ |
|--------|-------------|
| `aspforms/Services/IFileService.cs` | إضافة دالة جلب معلومات الملف بالاسم المخزَّن. |
| `aspforms/Services/FileService.cs` | تنفيذ البحث عن المرفق في قاعدة البيانات بالاسم المخزَّن. |
| `aspforms/Controllers/FilesController.cs` | إضافة مسار `GET .../api/Files/download/by-name/{fileName}`. |

### مشروع `forms_front` (Next.js)

| الملف | ماذا تغيّر؟ |
|--------|-------------|
| `forms_front/src/lib/attachment-url.ts` | **ملف جديد**: تطبيع روابط المرفقات القديمة نحو مسار التنزيل الصحيح. |
| `forms_front/src/app/api/files/download/[fileId]/route.ts` | **ملف جديد**: Proxy لتنزيل الملف بالمعرّف عبر الواجهة. |
| `forms_front/src/app/api/files/download/by-name/[fileName]/route.ts` | **ملف جديد**: Proxy لتنزيل الملف بالاسم عبر الواجهة. |
| `forms_front/src/components/submissions/SubmissionViewer.tsx` | استخدام تطبيع الرابط عند المعاينة والتصدير. |
| `forms_front/src/app/(dashboard)/responses/page.tsx` | استخدام تطبيع الرابط في المرفقات والجدول وتصدير PDF. |

---

## ملاحظة بعد النشر

يجب **إعادة بناء ونشر** الواجهة (`forms_front`) والـ API (`aspforms`) حتى تدخل التعديلات حيّز التنفيذ. إذا بقي الرابط يعيد 404 بعد النشر، قد يكون الملف غير موجود في قاعدة البيانات أو على القرص لهذا المعرّف.
